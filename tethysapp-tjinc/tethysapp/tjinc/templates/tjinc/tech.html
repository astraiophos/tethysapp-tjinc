    Software Design Document

    Technical Specifications

    Software Details

    This web page application was created in the Tethys app platform. It is used to display an interactive map of user-defined watersheds in Utah County. The app is written in HTML and Javascript and was built in a virtual machine (Oracle VM VirtualBox). It references a watershed delineation tool available on the BYU geoserver, which was created using ModelBuilder in ArcMap 10.3. The code was written in PyCharm editor. The web app is designed to run in the Tethys portal. If the tool is being run locally and not from the server, Tethys must be launched in the virtual machine terminal prior to opening the web browser to view the webpage. Contact either Jacob Fullerton or Trey McGhin to obtain the code at jacobbfull@gmail.com and creden791@gmail.com respectively. Documentation on the Tethys Platform can be found at http://docs.tethysplatform.org/en/latest/index.html.

    Compatibility and Limitations

    The web page is available for viewing in any web browser. For running locally, the code is designed to run in Linux. It can also be run on Mac OSX and Windows, but the installation is slightly different. See http://docs.tethysplatform.org/en/latest/installation.html for installation instructions. The machine requires at least a 64-bit operating system and 8.00 GB of RAM.

    Purpose

    The purpose of this code is to create a map that displays information conveying the spatial extent of watersheds in Utah County, with outlets defined by the user. The map also displays a base topographic map with computed flowlines for geo-referencing. The polygon of the resulting watershed is available for viewing to user once the outlet has been defined and the geoprocessing tool has completed.  Buttons are available to the right side of the window that allow the user to view any desired help files. This is to provide further clarification to the user on how the web page works.

    Main Files (what’s needed for user)

    For this website, the user is not required to provide any necessary files. All files required to run the geoprocessing tool within the web page have already been provided on the geoserver maintained by BYU.     User Interface

    Figure 4 shows the full webpage. To the side of the main window is the side menu, where the user may click on one of three buttons which links to either the main window, the user help file, or the technical specifications file. In the main window, the user defines the outlet of a desired watershed delineation by clicking on one of the visible flowlines. With the left-mouse click, an open point will indicate the outlet point. After a few minutes of processing, the delineated watershed will appear (Figure 4). This user interaction feature is used so the user can define any possible watershed or sub-basin at any outlet location.
